# 진행팀 Scheduler

교회 진행팀의 월간 섬김 가능 여부 제출과 라인업 관리를 위한 Google Apps Script Web App MVP입니다.

## 파일 구성

- `Code.gs`: Apps Script 백엔드, Google Sheets 구조 생성, CRUD, 제출 저장, 제출 현황, 라인업, 이메일 리마인더
- `Index.html`: 첫 진입 화면
- `Submission.html`: 찬양팀/진행팀 가능 주일 제출 페이지
- `Admin.html`: 관리자 화면 기본 구조
- `Members.html`, `Positions.html`, `Availability.html`, `Assignments.html`, `Settings.html`: 관리자 화면별 섹션
- `Styles.html`: 공통 스타일
- `Scripts.html`: 공통 프론트엔드 동작

## Google Apps Script 설정 방법

1. 새 Google Sheet를 만듭니다.
2. 상단 메뉴에서 `확장 프로그램 > Apps Script`를 엽니다.
3. 이 프로젝트의 파일들을 Apps Script 프로젝트에 같은 이름으로 만들고 내용을 복사합니다.
4. 프로젝트를 저장합니다.
5. Apps Script에서 `setupWorkbook` 함수를 한 번 실행합니다.
6. 권한 승인 화면이 나오면 승인합니다.
7. `배포 > 새 배포 > 웹 앱`으로 배포합니다.
8. 첫 MVP 공유 링크용으로는 `실행 사용자: 나`, `액세스 권한: 모든 사용자`로 배포합니다.
9. 생성된 배포 URL을 `Settings` 시트의 `Web_App_URL` 값에 입력합니다.

## 접속 주소

- 첫 진입 화면: 배포 URL
- 팀원 제출 페이지: 배포 URL 뒤에 `?page=submission` 추가
- 관리자 대시보드: 배포 URL 뒤에 `?page=admin` 추가

## Version 1 포함 기능

- 필요한 Google Sheet 탭과 헤더를 자동 생성합니다.
- 기본 포지션을 생성합니다: 자막, 조명, 음향, 카메라, 믹싱, PD, FD
- 팀원 제출 화면에는 활성 멤버만 표시됩니다.
- 제출 화면은 찬양팀/진행팀 탭을 제공하며, 하나의 Spreadsheet 안에서 진행팀은 `OP-` 접두어 탭, 찬양팀은 `PR-` 접두어 탭을 사용합니다.
- 이름을 선택하면 고정 포지션이 자동으로 표시됩니다.
- `Current_Target_Month` 값을 기준으로 해당 월의 주일을 자동 생성합니다.
- 같은 멤버가 같은 월에 다시 제출하면 기존 제출 행을 업데이트합니다.
- 멤버 관리에서 추가, 수정, 비활성화, 재활성화, 고급 삭제를 지원합니다.
- 포지션 관리에서 추가, 수정, 주일별 필요 인원 설정, 활성화, 비활성화, 삭제를 지원합니다.
- 활성 멤버 기준으로 제출 현황을 계산합니다.
- 라인업 표는 활성 포지션을 기준으로 동적 열을 생성합니다.
- 기본 라인업 생성은 제출된 가능 주일을 기준으로 포지션별 배정을 균등하게 분배합니다.
- 리마인더 이메일 기능은 미제출자에게 이메일을 보내고 결과를 `Email_Log`에 기록합니다.

## 참고 사항

- 이 MVP는 Google Sheets를 데이터베이스로 사용하므로 관리자 접근은 신뢰할 수 있는 사람에게만 제한하는 것을 권장합니다.
- 현재 버전에는 별도의 관리자 권한 시스템이 없습니다. 더 안전한 관리자 권한 관리는 다음 단계에서 추가하는 것을 권장합니다.
- 멤버를 비활성화해도 과거 가능 주일 제출 기록과 라인업 기록은 유지됩니다. 기본 운영 방식은 삭제보다 비활성화/재활성화를 권장합니다.
